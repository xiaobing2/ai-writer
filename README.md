## 📖 项目简介

**AI 文案工作台** 是一个基于阿里云通义千问大模型的智能文案生成工具。通过调用阿里云 DashScope API，帮助用户快速生成各类高质量文案，包括营销文案、产品介绍、社交媒体内容、邮件模板、创意故事等。

### 核心功能

- 🎯 **多种文案类型**：支持营销文案、产品介绍、社交媒体、邮件模板、创意故事、自定义提示词等
- 🤖 **AI 智能生成**：基于阿里云通义千问（Qwen）大模型，生成高质量文案内容
- 🎨 **现代化 UI**：简洁美观的用户界面，操作流畅，体验优秀
- 📋 **一键复制**：生成结果可快速复制到剪贴板，提高工作效率
- ⚡ **边缘加速**：部署在阿里云 ESA Pages，享受全球边缘节点加速

## 🛠️ 技术栈

- **前端框架**：Vue 3 + Vite
- **AI 模型**：阿里云通义千问（Qwen）系列模型
- **API 接口**：阿里云 DashScope OpenAI 兼容接口
- **部署平台**：阿里云 ESA Pages
- **样式方案**：原生 CSS + 响应式设计

## 🚀 快速开始

### 环境要求

- Node.js >= 18.0.0
- npm >= 8.0.0

### 安装步骤

1. **克隆项目**
```bash
git clone <your-repo-url>
cd ai-writer
```

2. **安装依赖**
```bash
npm install
```

3. **（必需）准备阿里云 API Key**

> 评审和使用者需要在页面中**手动输入自己的阿里云 DashScope API Key**，项目不会内置任何密钥。

- 登录 [阿里云百炼平台](https://dashscope.console.aliyun.com/)
- 进入「API-KEY 管理」，创建新的 API Key，并妥善保管
- 运行项目后，在左侧的「阿里云 DashScope 配置」区域粘贴该 Key

> 提示：仓库中附带的 `.env.example` 主要用于**本地开发时预填 BaseURL 和模型名**，不建议在 `.env` 中保存真实 API Key。

4. **启动开发服务器**
```bash
npm run dev
```

5. **构建生产版本**
```bash
npm run build
```

构建产物在 `dist` 目录，可直接部署到静态托管服务。

## 📦 部署到阿里云 ESA Pages

### 部署步骤

1. **准备代码仓库**
   - 将代码推送到 GitHub 公开仓库
   - 确保 `.env` 文件已添加到 `.gitignore`（不要提交敏感信息）

2. **在 ESA Pages 创建站点**
   - 登录 [阿里云 ESA 控制台](https://esa.console.aliyun.com/)
   - 进入「Pages」服务
   - 点击「新建站点」，选择「GitHub 仓库」
   - 连接你的 GitHub 账号并选择仓库

3. **配置构建设置**
   - **构建命令**：`npm install && npm run build`
   - **输出目录**：`dist`
   - **Node 版本**：选择 18.x 或更高

4. **（可选）配置环境变量**
   - 如需在构建时预填 BaseURL 或模型名，可在 ESA Pages 站点设置中添加环境变量：
     - `VITE_DASHSCOPE_BASE_URL`
     - `VITE_MODEL_NAME`
   - **API Key 不建议通过环境变量下发给前端**，而是由用户在页面左侧手动输入，便于比赛评审与安全管控。

5. **部署**
   - 点击「部署」按钮
   - 等待构建完成
   - 获取部署后的访问 URL

## 🎯 使用说明

1. **选择文案类型**：在左侧选择需要的文案类型（营销文案、产品介绍等）
2. **填写信息**：根据选择的类型，填写相应的表单信息
3. **生成文案**：点击「生成文案」按钮，等待 AI 生成结果
4. **复制使用**：生成完成后，点击「复制」按钮即可复制文案内容

## 🔒 安全说明

- **API Key 不会内置在仓库或前端代码中**，需要使用者在页面中手动输入
- 页面不会将 API Key 上传到任何后端，只在当前浏览器内存中使用，刷新页面即清空
- 如需更高安全性，可在后续版本中加入边缘函数（Edge Function）作为后端代理调用大模型

3. **网络环境**：确保能正常访问阿里云 DashScope API 服务
